# Task T-005: Remove duplicate UI dependencies

**Task ID:** T-005
**Feature:** FIX-002-performance
**Status:** Todo
**Priority:** Nice-to-have
**Depends on:** none

---

## 1. Context

`package.json` carries `daisyui` (the project standardized on shadcn/ui — daisyUI appears unused) and
both the `radix-ui` umbrella package **and** individual `@radix-ui/react-*` packages. Unused/duplicate
UI deps add install weight and risk shipping duplicate CSS/JS. This task removes what is genuinely
unused after verification.

Related requirements: RF-5 (see `00-performance-review.md`).

## 2. Description

Verify usage of `daisyui` and of the `radix-ui` umbrella vs the individual `@radix-ui/react-*`
packages, then remove whatever is unused and standardize on a single Radix import style. This is a
verify-then-prune task — nothing is removed without confirming zero references first.

## 3. Files to Touch

**Modify:**
- `client/package.json` — remove confirmed-unused deps
- `client/postcss.config.mjs` / `client/tailwind.config.ts` — drop the daisyUI plugin if present and unused
- `client/package-lock.json` — regenerated by the install after removal

**Read for context (no changes expected):**
- `client/src/ui-kit/ui/*` — confirm which Radix import style is actually used
- `client/app/globals.css` — confirm no daisyUI directives/classes in use

## 4. Implementation Plan

1. Grep the codebase for `daisyui`, `daisy`, and known daisyUI class patterns; confirm there are zero real usages.
2. Grep for `from 'radix-ui'` vs `from '@radix-ui/react-*'`; decide the single style to keep (prefer whichever the existing `ui-kit` components use).
3. Remove `daisyui` from `package.json` and any daisyUI plugin registration in the Tailwind/PostCSS config.
4. Remove the redundant Radix dependency style (either the umbrella `radix-ui` or the individual packages — whichever is unused).
5. Reinstall to refresh the lockfile, then run lint, typecheck, and the unit suite to confirm nothing broke.

## 5. Definition of Done

The task is complete when:

- [ ] Removed deps have zero references in `client/` (verified by grep before removal).
- [ ] `npm run lint`, `npm run typecheck`, and `npm run test` pass in `client/`.
- [ ] `npm run build` succeeds with no missing-module errors.
- [ ] No regressions in existing tests.
- [ ] Code follows existing project conventions and patterns.
- [ ] A single, consistent Radix import style remains across `ui-kit`.

## 6. Test Plan

**Unit tests (Vitest):** existing suite must stay green (regression guard); no new unit tests needed.

**Component tests (RTL):** existing component tests for Radix-based UI (dialog/select) must still pass.

**E2E tests (Playwright):** none required for this task.

**Test data needs:** none.

## 7. Notes & Considerations

- This is verify-first: if any dep turns out to be used, leave it and note the finding instead of forcing removal.
- Keep the change isolated to dependency hygiene — no component rewrites in this task.
